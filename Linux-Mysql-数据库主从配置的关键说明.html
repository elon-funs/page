Linux-Mysql-数据库主从配置的关键说明

andy发表: 

					#>0思路
		mysql 自带有bin-log 二进制日志（就是你在a机器上干过的,引起数据变动的事情,统统都搞成可以直接执行的程序,然后搞到别的机器上执行一遍,你说一样不一样？）
		所以需要让a,b机器建立关联。
			1.连接就是iP+端口,所以公网/内网/docker,能通就行,2.这段二进制程序,谁来生成,谁来执行。
			甚至二进制程序如何生成,怎么运行,怎么传输等等不属于配置问题了, mysql已实现。不用操心,想研究的请进入另一个主题
	
		#>1. 这台叫主服吧,给这台mysql起个名字server-id = 1 开启二进制日志,给日志起个文件名,存放路径也可以自己定,不写就默认

		echo "server-id=1
		log_bin=mysql-bin
		log_error=mysql-bin.err
		lower_case_table_names=1"
		>> /etc/mysql/conf.d/my.cnf

		#>2.在主服务器创建一个专门的同步用户,一般都是只读嘛
		GRANT REPLICATION SLAVE ON *.* TO "slave"@"%" IDENTIFIED BY "123456";
		
		#>3.建好了别乱动,一动就有日志了。是不是这个道理？走,搞从服务器

		#>4.在从服务器上加入对应的主服务器配置信息。
		前面有提到要分配server-id,别忘记了, 就加个server-id就行,别的用不上。
		CHANGE MASTER TO 
			MASTER_HOST="172.19.6.2",
		    MASTER_PORT=3306,
		    MASTER_USER = "slave",
		    MASTER_PASSWORD = "123456",
		    MASTER_LOG_FILE = "slave-mysql-bin.000001",
		    MASTER_LOG_POS = 430;
		
		#>5其他用到的命令
		mysql文件配置文件地址 /etc/mysql/conf.d/my.cnf					
		start slave;
		stop slave;//修改配置信息时, 需要先停止同步服务。
		show slave status 
		show master status 
		show grants for "slave"@"%"
		// 不需要同步哪些 -就在主服上设置,不产生日志,另一台就自然没了来源。
		binlog-ignore-db = mysql  
		binlog-ignore-db = test  
		binlog-ignore-db = information_schema  
		  
		// 同步哪些数据库,除此之外,其他不同步  
		binlog-do-db = game

		#>6特别提示
		两台数据库同时配置完成,再操作建库建表等数据操作.原因:就是不要在没同步前产生日志.

git-gitignore修改忽略文件

andy发表:2018-08-11 15:21:03 

    修改  .gitignore 把现在的ignore缓存得先改清掉，修改后再刷新ignore
    

    // .是目录当前下的所有,也可以单文件 
    git rm -r --cached .  

    //重新添加追踪
    git add .   
    git commit -m "msg" 
    git push origin master
    
    已在本地添加提交还没推到远程之前，如git add -f .　　gcam等操作后
    git reset --soft HEAD^
    这样就成功的撤销了你的commit　仅仅是撤回commit操作，您写的代码还在。

遇到这个种问题，一般都是网络原因
Failed to connect to github.com port 443:connection timed out(connection refused) 

git config --global http.proxy http://127.0.0.1:1080
 
git config --global https.proxy http://127.0.0.1:1080


取消代理

取消全局代理：
git config --global --unset http.proxy
